# Sports Network Portal — Demo Videos

Two narrated, brand-fidelity product tours of SNP. Same scene-player pattern
as the Oomfy investor demo (Kokoro-82M TTS, on-device, MIT-licensed).

| File | What it covers | Scenes | Duration |
|---|---|---|---|
| `demo-club.html` | The club web portal — all 9 modules, subscription, multi-tenant + multi-currency | 25 | ~9 min |
| `demo-mobile.html` | The mobile app — all 4 personas (player / coach / parent / fan), admissions, multi-language | 28 | ~10 min |

## What's in this folder

| File | Purpose |
|---|---|
| `demo-club.html` | Club web portal demo player. Open in Chrome → ▶ play. |
| `demo-mobile.html` | Mobile app demo player. Same pattern. |
| `script-club.json` | Source of truth — voice + per-scene narration for the club demo. Edit + re-run TTS to iterate. |
| `script-mobile.json` | Same, for the mobile demo. |
| `audio/club/scene-NN.mp3` | One narration MP3 per club scene (generated by `gen_audio_kokoro.py`). |
| `audio/mobile/scene-NN.mp3` | Same, for mobile demo. |
| `base.css` | Brand primitive library. Mirrors the actual SNP web + mobile widgets (browser chrome, phone shell, KPI tiles, bilingual rows, etc.). |
| `gen_audio_kokoro.py` | Local Kokoro-82M synth driver. Reads a script JSON, writes MP3s into `audio/<club\|mobile>/`. |

## First-time setup (Kokoro)

You already have Kokoro installed from the Oomfy demo. If not:

```bash
pip install kokoro soundfile numpy
# (Kokoro also requires espeak-ng on macOS: `brew install espeak-ng`)
```

## Generate the audio

```bash
cd /Users/harsh/Development/flutter_apps/snp/demo-video

# club demo audio → audio/club/scene-01.mp3 … scene-25.mp3
python3.11 gen_audio_kokoro.py --script script-club.json

# mobile demo audio → audio/mobile/scene-01.mp3 … scene-28.mp3
python3.11 gen_audio_kokoro.py --script script-mobile.json
```

Each script takes ~60-90 seconds to render the full set of MP3s on a modern Mac. The voice is `am_adam` (clean professional narrator) — change the `voiceCode` field in either script to swap.

## How to play

Open either `demo-{club,mobile}.html` in Chrome → click **▶ play full tour**. The first MP3 needs the click to satisfy autoplay policy; everything else advances automatically when each MP3 ends.

**Keyboard controls during playback**:
- `→` or `Space` — skip to next scene
- `←` — go back one scene
- `⌘ + ⌃ + F` — fullscreen (use this before screen-recording)

**Touch controls (phones / tablets)**:
- tap the right side of the screen — next scene
- tap the left side — go back
- swipe left / right — next / back

Both demos are mobile responsive. In landscape the canvas is the original
1280×720; in portrait it re-stacks onto a 720×1280 canvas (split scenes put
copy above the mock, browser KPIs re-grid 2×2, the sidebar narrows) — all
handled by the `@media (orientation: portrait)` layer in `base.css`. The
player JS reads the live canvas size, so nothing else needs to change.

**Preview a specific scene without audio**: append `?scene=N` to the URL, e.g.
`demo-club.html?scene=14` jumps straight to scene 14 and shows the static frame.

## How to record into MP4

The HTML *is* the video. To share it as MP4:

### Option A — QuickTime (no install)
1. Open `demo-club.html` (or `demo-mobile.html`) in Chrome.
2. ⌘+⌃+F for fullscreen.
3. QuickTime Player → File → **New Screen Recording**.
4. Pick the recording region (full window).
5. Hit ▶ Record, then click ▶ play in Chrome.
6. Wait ~10 min for the tour to complete, stop after the close slide fades.
7. File → Export As → MP4 (1080p).

### Option B — ffmpeg post-convert
```bash
ffmpeg -i snp-club-demo.mov -c:v libx264 -crf 18 -preset slow -c:a aac -b:a 192k snp-club-demo.mp4
```

## How to iterate

### Change the narration
1. Edit any scene's `narration` field in `script-club.json` or `script-mobile.json`.
2. Re-run `python3.11 gen_audio_kokoro.py --script script-<name>.json`.
3. Reload the HTML in Chrome.

### Change the voice
Edit `voiceCode` at the top of the script JSON. Kokoro voices:

- **`am_adam`** (current — clean professional male, narrator-tier)
- `am_michael` (deeper male)
- `am_eric` (warm male)
- `af_bella`, `af_sarah`, `af_sky`, `af_nicole` (female)
- `bm_george`, `bm_lewis` (British male — phonemizer is US English so accents render flat)

### Add a scene
1. Add a `scenes[]` entry to the script JSON with the next `order` number.
2. Re-run the Kokoro synth — it'll add the missing MP3.
3. Add a matching `<section class="scene" data-id="N" data-audio="audio/<sub>/scene-NN.mp3">` block to the HTML, reusing component classes from `base.css`.

## Brand primitives in `base.css`

The component library every scene composes from. Browser frames, phone shells, bilingual table rows, KPI tiles, pills, animations — all mirror the live SNP product.

| Class | What it's for |
|---|---|
| `.scene` / `.scene.active` | Scene wrapper; only one is `.active` at a time |
| `.cover-block` | Centered hero layout for cover / overview / close scenes |
| `.split` / `.split.flip` | Text-left + mock-right layout (most module spotlights) |
| `.browser` + `.browser-chrome` + `.browser-body` | Web portal mock |
| `.b-sidebar` + `.b-side-item.active` | Module list |
| `.b-main` + `.b-pageheader` + `.b-pagetitle` | Page chrome |
| `.b-kpis` + `.b-kpi` | KPI tile row |
| `.b-table` + `.b-thead` + `.b-row` + `.c1 .en + .ar` | Bilingual table rows (Arabic in `.ar`) |
| `.pill` (`.ok`, `.warn`, `.err`, `.info`, `.mute`) | Status pills |
| `.phone` + `.phone-screen` + `.ph-status` + `.ph-appbar` + `.ph-body` | Mobile mock |
| `.ph-stat-card`, `.ph-row`, `.ph-tabs`, `.ph-menu`, `.ph-attendance` | Mobile components |
| `.a-fadeup`, `.a-fade`, `.a-scale`, `.a-pulse-v` + `.d-1`..`.d-5` | Animations + stagger |

## Tip for live presentation

If you're walking a club through this in a meeting, just keep the HTML in
Chrome. `→` / `Space` lets you skip past a scene if the conversation has
moved on. `←` lets you replay a scene you breezed past.
